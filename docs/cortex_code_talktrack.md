# Cortex Code - Demo Talk Track

How to narrate Cortex Code so Shawnee leaves wanting it on her team.

## Anchor message
> "Every artifact you'll see today was generated by Cortex Code from a
> prompt contract that lives in this workspace. Same workspace, same
> contracts, will run on your trial instance tomorrow."

## Recurring beats (use one per vignette)

### Beat 0 - Open the workspace (once, at the start)
"Everything I'm about to do happens inside Snowsight. The left rail is
this Git repo, mounted as a Workspace. The right panel is Cortex Code.
Same surface Shawnee will use on her trial tomorrow." Open `RUN.md`.

### Beat 1 - Plan mode (open the demo)
"Notice we did not write any code yet. Cortex Code planned the demo
first. This is how you get reviewable AI: plan -> approve -> build."

### Beat 2 - `@`-mention contract (every vignette)
"I'm pointing Cortex Code at this contract with `@infrastructure/prompt-contract.md`. The YAML at the top tells it which role, warehouse, schema to target. The Markdown body is the spec."

### Beat 3 - Skills (V1, V2, V3, V5, V6)
"`/skill semantic-view` switches Cortex Code into a mode that knows our
semantic-view DSL, our reflect tool, and our verified-query workflow.
Skills are how we encode best practices."

### Beat 4 - Subagent (V4)
"I'll spawn the `sql-verify` subagent in the background while I keep
talking. This is how an SE multitasks safely. When it returns, it tells
me whether the join risks fanout."

### Beat 5 - Role-switch + multi-tenant (V5)
"I'm switching role to `OB_DEMO_LENDER_BIG` in Snowsight - same view,
filtered to high-volume TPOs. Now `OB_DEMO_LENDER_SMALL` - same view,
filtered to California. **One product. One row-access policy. N tenants.**
This is how Solution Center scales on Snowflake. And if I changed a TPO's
compliance_score in the producer side right now, both lenders would see
it on their next query. No FTP, no batch refresh."

### Beat 6 - The close
"Take this workspace. Open Cortex Code on your trial. Type
`@infrastructure/prompt-contract.md` and say 'recreate this in my
trial.' In about ten minutes you have what we just walked through."

## Snowsight surfaces to use

- **Worksheets** with Cortex Code side panel: V1, V2, V3, V4, V5 SQL.
- **Workspaces / Projects**: open `optimalblue-demo/` so all contracts and
  SQL are visible to Cortex Code via `@`-mentions.
- **Streamlit-in-Snowflake editor**: V6 - open `app.py` and let Cortex Code
  explain a chart block.
- **Role switcher** (top right): V5 producer/consumer split.

## Things to NOT skip

- Show one error fix live ("expand error" -> Cortex Code fix-it). Pick
  a typo in the V1 contract on purpose if needed.
- Show the inheritance pattern: open V3 contract, scroll to top, point
  at `inherits: ../../infrastructure/prompt-contract.md`. "Convention
  reuse, not copy-paste."
- Show one `system_todo_write` panel updating live. Reinforces the
  "AI you can audit" message.
